# Tomato Plant Disease Prediction API

AI-powered FastAPI service for detecting tomato plant diseases using a TFLite/Keras model, with an optional AI assistant for treatment advice.

## Quick Overview
- Service type: FastAPI microservice
- Primary model: `model/MultiBranchFusionNet-0.tflite` (preferred)
- Keras fallback: `model/MultiBranchFusionNet-0.keras`
- Optional validator: `model/tomato_not_tomato.keras` (checks if image contains a tomato plant)

## Features
- Predicts tomato diseases (10 classes) and returns class + confidence.
- Optional AI-generated treatment advice via Gemini (controlled by environment variables).
- Basic image quality checks and optional tomato validation.

## Repo structure (important files)
- `app.py` ŌĆö FastAPI application entrypoint and startup loader
- `api/predict.py` ŌĆö API routes for prediction and model info
- `service/predict_service.py` ŌĆö `Predictor` and `TomatoValidator` implementations
- `service/assistant_service.py` ŌĆö AI assistant wrapper (Gemini) for treatment advice
- `model/` ŌĆö directory for model files (TFLite / Keras / validator)
- `src/tflite_converter.py` ŌĆö helper to convert Keras -> TFLite
- `requirements.txt` ŌĆö Python dependencies

## Prerequisites
- Python 3.8+ recommended (match your TensorFlow version requirements)
- Install dependencies:

```bash
python -m venv .venv
source .venv/bin/activate   # use `.\.venv\Scripts\activate` on Windows
pip install -r requirements.txt
```

## Environment variables
Create a `.env` file at the project root (the code uses `python-dotenv`):

```env
# Optional: enable Gemini AI assistant
GEMINI_API_KEY=your_gemini_api_key_here
MODEL_NAME=your_gemini_model_name_here

# Example values: MODEL_NAME could be a textual model id used by your provider
```

- If `GEMINI_API_KEY` is not set, the AI assistant will be disabled and the service will fall back to built-in `disease_treatments`.

## Models
Place the pre-trained models into the `model/` directory. Expected filenames used by the service:
- `MultiBranchFusionNet-0.tflite` ŌĆö preferred runtime model (TFLite)
- `MultiBranchFusionNet-0.keras` ŌĆö Keras model used as a fallback and for conversion
- `tomato_not_tomato.keras` ŌĆö (optional) validator model to detect tomato vs non-tomato images

If you have only the Keras model, use the converter at `src/tflite_converter.py` to produce a TFLite file.

Convert Keras -> TFLite (from repo root):

```bash
python src/tflite_converter.py
```

Notes:
- TensorFlow conversion can be sensitive to TF version and available memory. If conversion fails, the script includes fallback attempts (SavedModel intermediate, select TF ops, etc.).

## Running the service
- Development (auto-reload):

```bash
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

- Or run directly:

```bash
python app.py
```

The API docs will be available at `http://localhost:8000/docs` when running.

## API Reference

Base router prefix: `/predict_disease`

- `GET /predict_disease/` ŌĆö Welcome message
- `GET /predict_disease/health` ŌĆö Health check, returns `{ "status": "ok" }`
- `POST /predict_disease/predict` ŌĆö Main endpoint for disease prediction

POST `/predict_disease/predict` parameters:
- `file` (multipart/form-data, required) ŌĆö image file
- `include_advice` (query, bool, default: `false`) ŌĆö whether to include AI advice
- `region` (query, optional) ŌĆö farmer region to tailor advice
- `language` (query, default: `en`) ŌĆö language for AI advice

Example cURL (include AI advice):

```bash
curl -X POST "http://localhost:8000/predict_disease/predict?include_advice=true&region=Kenya&language=en" \
  -H "Content-Type: multipart/form-data" \
  -F "file=@/path/to/image.jpg"
```

Python example using `requests`:

```python
import requests

files = {"file": open("/path/to/image.jpg", "rb")}
params = {"include_advice": "true", "region": "Kenya", "language": "en"}
resp = requests.post("http://localhost:8000/predict_disease/predict", files=files, params=params)
print(resp.status_code, resp.json())
```

Model info endpoint:
- `GET /predict_disease/model/info` ŌĆö returns basic info about loaded model and assistant status

## Example response

Successful response (typical):

```json
{
  "disease": "Tomato___Early_blight",
  "confidence": 0.937,
  "treatment": "Remove lower infected leaves, apply fungicide, and mulch around plants.",
  "success": true
}
```

- When `include_advice=true` and `GEMINI_API_KEY` is set, the `treatment` field will be generated by the AI assistant.
- Otherwise `treatment` comes from the static `disease_treatments` dictionary in `service/predict_service.py`.

## Integration options
1) Run as a standalone microservice and call the HTTP endpoints from your app.
2) Embed the predictor in-process by importing `prediction_service` from `service.predict_service`:

```python
from PIL import Image
from pathlib import Path
from service.predict_service import Predictor

p = Predictor(Path("model/MultiBranchFusionNet-0.tflite"), Path("model/MultiBranchFusionNet-0.keras"), Path("model/tomato_not_tomato.keras"))
p.load_model()
img = Image.open("/path/to/image.jpg").convert("RGB")
result = p.predict_from_image(img)
print(result)
```

Notes when embedding:
- Ensure model file paths are correct (you can pass absolute paths).
- `Predictor.load_model()` is idempotent ŌĆö it will load TFLite first and fallback to Keras if needed.

## Known issues & recommended fixes
- Bug: in `api/predict.py` the variable `disease` is referenced in the `elif` branch even when `include_advice` is `False`. This will raise `NameError`. Fix by extracting `disease` and `confidence` before the conditional:

```python
# top of endpoint, after getting `result`
disease = result.get("disease", "Unknown")
confidence = result.get("confidence", 0.0)

if include_advice and result.get("success"):
    # use disease/confidence
    ...
else:
    # use disease/confidence
    ...
```

- Bug: in `service/assistant_service.py` the healthy-check uses `if "Healthy Plant" in disease.lower():` which will never match because `disease.lower()` is lowercase. Use `if "healthy plant" in disease.lower():` or perform the check on the original-case string.

- Conversion and TF issues: The converter script takes multiple conversion attempts; if conversion fails, check your TensorFlow version and available RAM. Consider running conversion on a machine with more memory or using remote/cloud conversion.

## Troubleshooting
- 400: non-image file upload ŌĆö ensure `Content-Type` is an image and pass a valid image file
- 500: "Model file not found" ŌĆö ensure `model/` contains the required files and correct filenames
- If AI assistant returns errors, confirm `GEMINI_API_KEY` and `MODEL_NAME` are set and that `google.generativeai` is installed

## Contributing / Extending
- To add new classes, retrain a model and adjust `class_names` in `service/predict_service.py`.
- To change input size or preprocessing, update `Predictor.preprocess()` accordingly.
